Prime Factorization

Naive Algorithm‚Äã

Assume that we generated prime numbers using the Eratosthenes's Sieve. If n is in the "prime-generated" range, we can actually do prime factorization in O(logn). Make another array. While we are doing the Sieve, for composite numbers, put "the first prime that verified that this number is composite" and for prime numbers, put itself. This is easy to implement.

Then we can start with n, and continue to divide the prime numbers in the array.

If not, we can just check through all primes less than n‚Äã and divide by those primes until we can't. If all the primes multiply up to n, we are done. If not, there is exactly one prime more than n‚Äã that divides n.
